# UX 体验修复 v0.5

## 解决的问题

| # | 问题 | 当前行为 | 目标行为 |
|---|------|---------|---------|
| 1 | 无障碍权限强依赖 | 不给无障碍不显示悬浮球 | 无障碍为可选项，不给也能用悬浮球手动点开 |
| 2 | 悬浮球只认 QQ | 硬编码只在 QQ 前台时显示 | 根据启用的输出目标动态识别（微信/QQ 前台都显示） |
| 3 | 缺少插件管理 UI | 输入源/输出目标全靠代码配置 | 主页面展示并管理启用的输入源和输出目标，支持导出/导入输出配置 JSON |
| 4 | 悬浮球出现时机不对 | 进入 QQ 就显示 | 用户点开输入框后才显示（有 Accessibility 时） |
| 5 | 边缘收缩 | 无 | 可选项：悬浮球拖到屏幕边缘自动收缩成小圆点 |
| 6 | 拖动 vs 点击混淆 | 当前难以区分点击和拖动 | 短按→打开表情窗口，长按→拖动位置 |

## 方案

### 1. 无障碍权限可选项

- `BubbleService.kt`: 增加 fallback 模式。无权限时悬浮球仍然显示（不依赖 AccessibilityEvent），点击直接跳到 StickerPickerActivity
- 主页面检测无障碍状态，若未启用则提示「建议开启以获得自动触发体验」，但不阻止使用
- **涉及文件:** `BubbleService.kt`, `MainActivity.kt`

### 2. 动态识别前台应用

- 新增 `ForegroundAppMonitor` 类，读取已启用的 `output-targets` 中所有 `packageName`
- 监听前台进程变化（`UsageStatsManager` 或 `AccessibilityService.onAccessibilityEvent(TYPE_WINDOW_STATE_CHANGED)`）
- 当前台 App 的包名匹配任一启用目标的包名 → 显示悬浮球
- **涉及文件:** 新建 `ForegroundAppMonitor.kt`, 修改 `QQDetectAccessibilityService.kt` → 改名为通用的 `AppDetectService.kt`

### 3. 插件管理 UI

- 主页面增加「输入源」和「输出目标」两个卡片
- 输入源：列出已安装的插件 APK，显示启用/禁用状态
- 输出目标：从 `ShareTargetLoader` 加载，可勾选启用/禁用；增加「导出 JSON」和「导入 JSON」按钮
- **涉及文件:** `MainActivity.kt`, 新建 `SettingsPage.kt` 或嵌入 Compose

### 4. 输入框检测显示悬浮球

- 在 `onAccessibilityEvent(TYPE_WINDOW_STATE_CHANGED)` 基础上，增加 `TYPE_VIEW_FOCUSED` 事件
- 检测到 EditText/输入框获得焦点 → 显示悬浮球
- 检测到输入框失去焦点或键盘收起 → 隐藏悬浮球（延迟几秒防闪烁）
- **涉及文件:** `AppDetectService.kt`（原 `QQDetectAccessibilityService.kt`）

### 5. 边缘收缩（可选项）

- 在 `BubbleService.kt` 中监听悬浮球位置
- 当 X 坐标靠近屏幕左/右边距 < 阈值（如 20dp）→ 缩小为小圆点（宽高减半）
- 点击收缩状态的小圆点→展开恢复正常大小
- 设置页增加开关「自动收缩到边缘」
- **涉及文件:** `BubbleService.kt`, `MainActivity.kt`

### 6. 长按拖动

- 默认 `onTouch`：ACTION_DOWN + ACTION_UP（短时间）→ 视为点击，打开表情窗口
- 长按（> 300ms）→ 进入拖动模式
- 或者：手指在悬浮球上滑动超过阈值才拖动，否则视为点击
- **涉及文件:** `BubbleService.kt`

## 不动摇的不变量

1. 悬浮球的 `TYPE_APPLICATION_OVERLAY` 权限仍需在运行时申请
2. QQDetectAccessibilityService → AppDetectService 重命名不影响 ADB 通信（Android:name 在 manifest 中更新即可）
3. 输出目标 JSON 格式不变（兼容用户手动编辑）
4. 输入源的插件 IPC 协议不变（AIDL interface 不修改）

## 影响评估

| 文件 | 改动量 |
|------|--------|
| `BubbleService.kt` | 大（拖动逻辑重写 + 边缘收缩 + 非无障碍 fallback）|
| `QQDetectAccessibilityService.kt` | 中（重命名为 AppDetectService + 动态识别 + 输入框检测）|
| `MainActivity.kt` | 中（插件管理 UI 卡）|
| 新建 `ForegroundAppMonitor.kt` | 小 |
| 新建 `SettingsPage.kt` | 中 |
| `AndroidManifest.xml` | 小（service 重命名）|
